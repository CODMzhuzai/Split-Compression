# 安全政策

## 报告安全漏洞

如果您发现了任何安全漏洞，请通过以下方式联系我们：

### 联系方式
- **Email**: [2487587079@qq.com](mailto:2487587079@qq.com)

### 报告格式
请在报告中包含以下信息：
- 漏洞的详细描述
- 复现步骤
- 受影响的版本
- 潜在的影响范围
- 可能的修复方案（如果有）

## 漏洞响应流程

1. **确认接收**：我们将在72小时内确认收到您的漏洞报告
2. **初步评估**：我们将在96小时内完成对漏洞的初步评估
3. **详细分析**：我们将在1周内完成对漏洞的详细分析
4. **修复开发**：我们将尽快开发修复方案
5. **测试验证**：修复方案将经过严格测试，确保不会引入新的问题
6. **发布更新**：修复方案将随下一个版本发布，我们将在14天内发布修复版本
7. **公开披露**：在修复发布后，我们将在14天内公开披露漏洞信息，包括修复版本号和详细的修复说明

## 支持的版本

我们只为最新的稳定版本提供安全更新。请确保您始终使用最新版本的分卷压缩工具。

| 版本 | 支持状态 |
|------|----------|
| 最新版 | ✅ 支持 |
| 旧版本 | ❌ 不支持 |

## 安全更新

本项目包含以下安全特性：

- 多线程设计避免UI阻塞
